Actúa como un **arquitecto senior de bases de datos especializado en PostgreSQL y sistemas de reservas de aerolíneas**, con conocimiento en sistemas similares a **Avianca, LATAM y plataformas de reserva como Despegar**.

Debes generar **un script SQL completo, limpio y ejecutable en PostgreSQL** que cree y pueble una base de datos para un **sistema web académico de gestión de reservas de una aerolínea**.

El diseño debe seguir estrictamente el alcance del siguiente proyecto universitario de **Ambiente Web I**, por lo que debes **respetar exactamente las entidades, relaciones y funcionalidades descritas**.

---

# OBJETIVO DEL SCRIPT

Construir una base de datos que soporte una aplicación web que permita:

* Registrar y gestionar clientes
* Gestionar vuelos
* Realizar reservas
* Comprar tiquetes
* Adquirir paquetes turísticos asociados al destino
* Administrar estados e historial de reservas
* Generar reportes comerciales y operativos

La base de datos debe ser **normalizada, clara y lista para integrarse con una aplicación web bajo arquitectura MVC**.

---

# REGLAS OBLIGATORIAS

1. El script debe ejecutarse **sin errores en PostgreSQL**.
2. Usar:

* SERIAL para claves primarias
* FOREIGN KEY
* NOT NULL
* CHECK cuando aplique
* nombres de tablas en singular
* nombres de columnas claros en español

3. Incluir **comentarios SQL explicando las decisiones importantes**.

4. No agregar módulos fuera del alcance del proyecto.

NO incluir cosas como:

* gestión de aeronaves
* tripulación
* mantenimiento
* inventario de asientos complejo
* programas de millas
* sistemas de pagos externos

El modelo debe mantenerse **simple y académico**.
